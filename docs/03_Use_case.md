# 03. 활용 시나리오 (Use Cases)

본 문서는 **confound-insight** 프로젝트가  
어떤 문제 상황에서, 어떤 사용자에게, 어떤 가치를 제공하는지를  
구체적인 활용 시나리오 중심으로 설명한다.

본 문서는 기능 나열이 아닌,  
**“왜 이 시스템이 필요한가”**에 대한 답을 제공하는 것을 목표로 한다.

---

## 1. 본 시스템이 다루는 문제 정의

의약품 및 화학 도메인에서는 다음과 같은 문제가 반복적으로 발생한다.

- 제품명은 존재하지만, **분석 가능한 화합물 단위 정보로 바로 연결되지 않는다.**
- 동일 성분임에도 불구하고, 명칭·표기·언어 차이로 인해 **데이터 단절이 발생한다.**
- 복합제 의약품의 경우, **각 성분을 분리하여 분석하기가 어렵다.**
- 화학 구조 기반 분석을 위해 필요한 데이터는 흩어져 있으며, 재사용이 어렵다.

본 시스템은 위 문제를  
**“제품명 → 주성분 → 화합물”이라는 일관된 흐름으로 해소**하는 것을 목표로 한다.

---

## 2. 주요 활용 시나리오

### 2.1 단일 성분 의약품 분석 시나리오

#### 상황
- 사용자는 특정 의약품 제품명을 알고 있으나,
  해당 제품의 화학적 특성이나 구조 정보에 접근하기 어렵다.

#### 처리 흐름
1. 사용자는 의약품 제품명을 입력한다.
2. 시스템은 제품명에 해당하는 주성분을 정규화한다.
3. 정규화된 주성분을 기준으로 PubChem 화합물 정보를 수집한다.
4. 화합물 구조 및 물성 정보를 분석 결과로 제공한다.

#### 제공 가치
- 제품명을 화합물 단위 분석으로 연결하는 **명확한 경로 제공**
- 화학 구조 기반 정보에 대한 **접근 장벽 해소**
- 반복 가능한 분석 파이프라인 확보

---

### 2.2 복합제 의약품 분석 시나리오

#### 상황
- 하나의 제품에 여러 주성분이 포함된 복합제 의약품의 경우,
  각 성분을 개별적으로 분석하기가 어렵다.

#### 처리 흐름
1. 사용자는 복합제 의약품 제품명을 입력한다.
2. 시스템은 해당 제품에 포함된 주성분 목록을 도출한다.
3. 각 주성분을 **독립된 화합물 단위로 분리**하여 처리한다.
4. 성분별 구조 정보 및 유사도 분석 결과를 개별 제공한다.

#### 제공 가치
- 복합제 의약품을 **성분 단위로 분해하여 분석**
- 특정 성분에 대한 구조적 특성만을 선택적으로 확인 가능
- 제품 중심이 아닌 **화합물 중심 사고 전환 지원**

---

### 2.3 성분 식별 불확실성 대응 시나리오

#### 상황
- 일부 성분은 표준 명칭이 불명확하거나,
  PubChem에서 즉시 식별되지 않을 수 있다.

#### 처리 흐름
1. 제품명 정규화 단계에서 원문 성분명을 함께 저장한다.
2. CID 조회 실패 시에도 성분 레코드는 유지한다.
3. 해당 성분은 배치 처리 대상 또는 후속 검증 대상으로 관리한다.

#### 제공 가치
- 데이터 누락 없이 **불완전한 상태도 관리 가능**
- 재처리 및 검증이 가능한 구조 확보
- 데이터 파이프라인의 **내구성 및 재현성 강화**

---

### 2.4 구조 기반 비교·탐색 활용 시나리오

#### 상황
- 사용자는 특정 성분과 구조적으로 유사한 화합물 정보를 확인하고자 한다.
- 효능이나 임상적 판단이 아닌, **순수 화학 구조 비교**가 목적이다.

#### 처리 흐름
1. 기준 화합물의 구조 정보를 기반으로 분자 지문을 생성한다.
2. 저장된 화합물 데이터와의 구조 유사도를 계산한다.
3. 유사도 점수를 기준으로 결과를 정렬하여 제공한다.

#### 제공 가치
- 화합물 구조 간 관계를 **정량적 수치로 비교**
- 재사용 가능한 유사도 분석 결과 확보
- 추가 분석 및 연구의 출발점 제공

---

## 3. 본 시스템의 활용 범위

본 시스템은 다음 용도로 활용될 수 있다.

- 화합물 중심 데이터 파이프라인 설계 예시
- 제약·화학 도메인 백엔드 시스템 설계 사례
- 외부 API 의존성을 관리하는 구조적 접근 방식 참고
- 구조 데이터 기반 분석 로직의 기술적 데모

---

## 4. 의도적으로 포함하지 않은 활용

본 시스템은 다음을 목적으로 하지 않는다.

- 약효 비교
- 안전성 평가
- 대체 의약품 추천
- 임상적 의사결정 지원

이는 본 시스템이 **분석 도구이지 판단 도구가 아님**을 명확히 하기 위함이다.

---

## 5. 본 문서의 핵심 메시지

confound-insight는  
**“제품명이라는 불완전한 입력을, 분석 가능한 화합물 단위로 정규화하는 파이프라인”**이다.

---

## 6. 향후 확장 가능성

- 신규 화학 데이터 소스 연동
- 구조 비교 기준 다양화
- 분석 결과의 시계열 관리
- 데이터 품질 메트릭 도입
